# PF-090 — Implement Robust Deduplication & File Hashing

> **Status:** Completed
> **Phase:** 3 — Reliability & Data Integrity
> **Objective:** Prevent duplicate data using a tiered strategy: File Hashing -> Gold Standard (Statement Balance) -> Regular Composite Key.

## Objective

Enhance the data import pipeline to ensure zero duplicates. This involves a tiered verification process:
1.  **File Hashing**: Block exact same files from being processed twice.
2.  **Tiered Transaction Deduplication**:
    *   **Gold Standard**: Use `Statement Balance` as part of the unique key for high-fidelity matching.
    *   **Migration Fallback**: If either the new or existing transaction lacks a balance (e.g. legacy data), fall back to a `Regular Composite Key` to ensure duplicates are caught even across system upgrades.

Update `TransactionService.FilterOutDuplicatesAsync` to implement the tiered strategy.

**Logic Flow:**
1.  **Standardize Keys**: All keys use UTC dates and fixed 2-decimal-place strings for amounts.
2.  **Existing Data**: Build sets for both `FullKeys` (with balance) and `RegularKeys` (without balance).
3.  **Tiered Check**:
    *   If New has balance: Match against `FullKeys` OR match against `RegularKeys` (where DB has no balance).
    *   If New lacks balance: Match against `RegularKeys`.
4.  **Intra-batch**: Keep track of seen keys within the current upload to block duplicates in the same file.
