 # Learning Guide: PF-009 — Hello LLM

## What PF-009 Is

Based on the [ROADMAP](ROADMAP.md) and [CLAUDE.md](CLAUDE.md), PF-009 is your first real LLM integration: replacing/augmenting the brittle bank parsers with Claude API. You'll build `LlmTransactionExtractor` — a C# class in `Infrastructure/Ai/` that sends raw bank statement text to Claude and gets back structured JSON transactions.

Your starting point is excellent. You already understand:
- **Clean Architecture**: where the new component lives
- **MediatR/CQRS**: how it plugs into the pipeline
- **IBankStatementParser strategy pattern**: what you're extending
- **TransactionDto**: what Claude needs to return

**What's new to you**: LLM APIs, prompt design, token costs, structured output.

### Key concepts to anchor in your mental model:

| Term | What it maps to in your world |
| :--- | :--- |
| **Prompt** | Your HTTP request body |
| **Completion/Response** | The HTTP response body |
| **Temperature** | Randomness (0 = deterministic, like a database; 1 = creative) |

| **System prompt** | Like middleware — sets the AI's "role" before your message |

## Layer 3 — Prompt Engineering for Structured Output

**Goal**: Get Claude to reliably return the exact JSON shape of `TransactionDto`.

This is the most important skill for PF-009. A bad prompt = unparseable responses.

### The structured output pattern

**System**:
```text
You are a bank statement parser.
Extract ALL transactions from the text below.
Return ONLY a valid JSON array. No markdown, no explanation.

Each item must match this exact shape:
{
  "date": "2025-01-14",       // ISO 8601
  "description": "GRAB FOOD",
  "flow": "DB",               // "DB" = debit/expense, "CR" = credit/income
  "amountIdr": 45000,         // always positive
  "balance": 1255000          // null if not available
}

If you cannot parse a line, skip it. Never invent data.
```

**User**:
`[raw bank statement text here]`

### Visual resource: Anthropic's own prompt engineering guide

[docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/overview](https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/overview)

The most useful sections for PF-009:
- **"Be clear and direct"** — remove ambiguity from your prompt
- **"Use XML tags"** — Claude responds better when you wrap inputs in `<bank_statement>` tags
- **"Prefilling Claude's response"** — start Claude's response with `[` to force JSON array output

### XML tags pattern (highly effective):

**User**:
```xml
<bank_statement>
14 Jan  GRAB FOOD          -45,000    1,255,000
15 Jan  TRANSFER IN SALARY +12,500,000 13,755,000
</bank_statement>

Return transactions as JSON array:
```

Then prefill the assistant's first token with `[` — this locks Claude into JSON mode.

## Layer 4 — Build It in PF-009

### What you're building:

```text
api/src/PersonalFinance.Infrastructure/
  Ai/
    ILlmTransactionExtractor.cs    ← interface (owned by Application)
    LlmTransactionExtractor.cs     ← implementation
    LlmParserPrompts.cs            ← prompt constants
```

### How it fits your existing architecture:

1.  `BankIdentifier.IdentifyAsync()` returns `null` (unknown bank)
2.  `CsvTransactionParser` falls back to `LlmTransactionExtractor`
3.  Sends raw text to Claude API
4.  Deserializes JSON → `List<TransactionDto>`
5.  Same pipeline continues (CategoryRule matching etc.)

### The interface you'll define (goes in `Application/Interfaces/`):

```csharp
public interface ILlmTransactionExtractor
{
    Task<List<TransactionDto>> ExtractAsync(string rawText, CancellationToken ct = default);
}
```

### The implementation skeleton:

```csharp
public class LlmTransactionExtractor(AnthropicClient client, IOptions<AiSettings> options)
    : ILlmTransactionExtractor
{
    public async Task<List<TransactionDto>> ExtractAsync(string rawText, CancellationToken ct)
    {
        var response = await client.Messages.GetClaudeMessageAsync(new MessageParameters
        {
            Model = options.Value.Model,
            MaxTokens = options.Value.MaxTokens,
            System = LlmParserPrompts.TransactionExtractionSystem,
            Messages = [
                new Message
                {
                    Role = RoleType.User,
                    Content = $"<bank_statement>{rawText}</bank_statement>\n\nReturn JSON:"
                },
                // Prefill assistant with "[" to force JSON array
                new Message { Role = RoleType.Assistant, Content = "[" }
            ]
        }, ct);

        var json = "[" + response.Content[0].Text; // re-add the prefilled "["
        return JsonSerializer.Deserialize<List<TransactionDto>>(json)!;
    }
}
```

## Key Gotchas to Watch For

1.  **JSON parsing failure**: Claude occasionally adds commentary before/after JSON. Use prefilling (`[`) + try/catch with logging.
2.  **Cost awareness**: A 5-page PDF statement ≈ ~3K tokens ≈ <$0.01. Fine for personal use.
3.  **The appsettings pattern**: Store ApiKey in environment variable, never in source. Add to `.env.example`.
4.  **Clean Architecture compliance**: The interface `ILlmTransactionExtractor` must live in `Application/Interfaces/`, not `Infrastructure/`. This is an existing `ARCH-02` rule in `governance.md`.
5.  **AnthropicClient DI registration**: Register as `AddSingleton` (it's thread-safe and manages its own HTTP client).
