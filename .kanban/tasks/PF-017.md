## Objective

Enable users to query transactions using natural language. "How much did I spend on food in January?" → LLM translates to a database query, executes, and returns a human-readable answer.

## Acceptance Criteria

- [ ] `POST /api/query` endpoint accepts natural language question
- [ ] LLM translates question into query parameters (date range, category, etc.)
- [ ] Query executed against transaction database
- [ ] LLM generates human-readable answer from results
- [ ] Supports: spending by category, date range filtering, comparisons
- [ ] Returns both the answer and the underlying data

## Dependencies

- [PF-015](PF-015.md) — LLM provider abstraction
- [PF-014](PF-014.md) — Transactions persisted in DB

## Learning Objectives

- Text-to-SQL / text-to-query patterns
- LLM as a query interface layer
- Prompt engineering for query decomposition
- Grounding LLM answers in real data
