## Objective

Set up the frontend test infrastructure (Vitest + React Testing Library) and write initial tests for API client functions and key components. Currently zero frontend tests exist.

## Acceptance Criteria

- [ ] `vitest`, `@testing-library/react`, `@testing-library/jest-dom`, `@testing-library/user-event` installed as dev dependencies
- [ ] `vitest.config.ts` (or inline in `vite.config.ts`) configured with `jsdom` environment
- [ ] `src/test/setup.ts` with `@testing-library/jest-dom` matchers imported
- [ ] `npm test` (or `npm run test`) script runs Vitest
- [ ] `transactionsApi.test.ts` — `getTransactions` called with correct URL, returns mapped data (mock `fetch`)
- [ ] `categoryRulesApi.test.ts` — `getCategoryRules` called with correct URL (mock `fetch`)
- [ ] `CashFlowDashboard.test.tsx` — renders without crashing with mock React Query data
- [ ] `TransactionUpload.test.tsx` (or equivalent upload component) — file input renders, empty submit shows error
- [ ] All tests pass via `npm test`
- [ ] `npm run build` still passes

## Technical Notes

- Vitest config in `vite.config.ts`:
```ts
test: {
  environment: 'jsdom',
  setupFiles: ['./src/test/setup.ts'],
  globals: true
}
```
- Mock `fetch` with `vi.fn()` — do not make real HTTP calls in tests
- Mock React Query with `QueryClientProvider` wrapping test renders
- Do NOT edit files in `src/components/ui/` — only test business components
- Keep tests colocated: `src/api/transactionsApi.test.ts`, `src/components/CashFlowDashboard.test.tsx`

## Dependencies

- None (no backend needed — all API calls mocked)
