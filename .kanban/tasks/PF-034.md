## Objective

Write unit tests for all MediatR command handlers and FluentValidation validators. Currently only `CategoryRuleService` has tests — handlers and validators have zero coverage.

## Acceptance Criteria

- [ ] `CreateCategoryRuleCommandHandlerTests` — happy path, validation failure, DB persist verified
- [ ] `UpdateCategoryRuleCommandHandlerTests` — existing rule updated, non-existent id returns null
- [ ] `DeleteCategoryRuleCommandHandlerTests` — existing rule deleted, non-existent id returns false
- [ ] `CreateTransactionCommandHandlerTests` — happy path, validation failure, domain event published
- [ ] `CreateCategoryRuleCommandValidatorTests` — missing keyword, type, category each produce correct error
- [ ] `CreateTransactionCommandValidatorTests` — missing required fields produce correct errors
- [ ] All tests use `UseInMemoryDatabase(Guid.NewGuid().ToString())` + `IDisposable` pattern
- [ ] Test naming follows `MethodName_Condition_ExpectedResult`
- [ ] `dotnet test` passes with all new tests green

## Technical Notes

- Follow the established pattern in `CategoryRuleServiceTests.cs`
- Use `Moq` for `IMediator` where needed; use real validator instances (no mocking)
- Handler tests should inject a real `AppDbContext` (in-memory) and real `IValidator<T>`
- Validator tests: instantiate validator directly, call `.Validate()`, assert `IsValid` and error messages
- File locations:
  - `tests/PersonalFinance.Tests/Commands/CreateCategoryRuleCommandHandlerTests.cs`
  - `tests/PersonalFinance.Tests/Commands/UpdateCategoryRuleCommandHandlerTests.cs`
  - `tests/PersonalFinance.Tests/Commands/DeleteCategoryRuleCommandHandlerTests.cs`
  - `tests/PersonalFinance.Tests/Commands/CreateTransactionCommandHandlerTests.cs`
  - `tests/PersonalFinance.Tests/Validators/CreateCategoryRuleCommandValidatorTests.cs`
  - `tests/PersonalFinance.Tests/Validators/CreateTransactionCommandValidatorTests.cs`
