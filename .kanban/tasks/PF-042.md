## Objective

Survey the MCP (Model Context Protocol) server ecosystem and identify which servers would provide the most value for this project's development workflow. The project already has access to Gmail and Google Calendar MCP servers via Claude Code. Goal is to document what's available, what's already wired up, and propose concrete integrations — e.g. a GitHub MCP for PR/issue management, a PostgreSQL MCP for direct DB introspection, or a filesystem MCP for richer file operations.

---

## Acceptance Criteria

- [ ] Inventory of MCP servers currently connected to this Claude Code instance (from available-deferred-tools)
- [ ] For each connected server: document what tools it exposes and a concrete use case for this project
- [ ] Research notable community/official MCP servers not yet connected (GitHub, PostgreSQL, Slack, Linear, Brave Search, etc.)
- [ ] Prioritized shortlist (top 3–5) of servers worth adding, with rationale per server
- [ ] Document findings in `docs/mcp-servers.md`
- [ ] Update `CLAUDE.md` or memory files if any server changes how we should work day-to-day

## Technical Notes

### Already connected (observed in Claude Code session)

| Server | Tools Available |
|--------|----------------|
| Gmail | `gmail_create_draft`, `gmail_get_profile`, `gmail_list_drafts`, `gmail_list_labels`, `gmail_read_message`, `gmail_read_thread`, `gmail_search_messages` |
| Google Calendar | `gcal_create_event`, `gcal_delete_event`, `gcal_find_meeting_times`, `gcal_find_my_free_time`, `gcal_get_event`, `gcal_list_calendars`, `gcal_list_events`, `gcal_respond_to_event`, `gcal_update_event` |

### Candidates to research

- **GitHub MCP** — create/close issues, review PRs, manage labels → replaces manual `gh` CLI usage
- **PostgreSQL MCP** — query DB schema and data directly → useful for ad-hoc debugging and migration validation
- **Filesystem MCP** — extended file ops (move, copy, tree listing) → improves file management in Claude
- **Brave Search / Exa** — web search grounded in live data → better than hallucinating docs
- **Linear MCP** — if we ever migrate kanban from flat files
- **Slack MCP** — team notifications on PR/deploy events

### Configuration

MCP servers are configured in Claude Code's settings (typically `~/.claude/claude_desktop_config.json` or via `claude mcp add`). Check `claude mcp list` to see active servers.
