## Objective

Send messy text (bank transaction lines) to an LLM and get back structured JSON. Learn prompt engineering for reliable structured output. Maps to **Day 2** of the learning schedule.

## Acceptance Criteria

- [ ] Script sends raw transaction text to LLM
- [ ] LLM returns structured JSON: `{ date, description, amount, currency, type }`
- [ ] JSON output is consistent across multiple runs (low temperature)
- [ ] System prompt enforces output schema
- [ ] Handles edge cases: missing fields, ambiguous amounts
- [ ] Pydantic model validates the LLM output

## Technical Notes

# Prompt engineering approach:

# 2. Use temperature=0 for deterministic output

Extract transaction data and return ONLY valid JSON.
