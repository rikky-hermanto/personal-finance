## Objective

Instrument all AI calls with observability: token usage, latency, cost per query, error rates. Dashboard for monitoring AI spend and performance.

## Acceptance Criteria

- [ ] Every LLM call logs: model, input_tokens, output_tokens, latency_ms, cost_usd
- [ ] Metrics endpoint or structured logs for aggregation
- [ ] Cost dashboard: daily/weekly/monthly AI spend
- [ ] Latency percentiles (p50, p95, p99) per endpoint
- [ ] Alerting threshold for cost anomalies
- [ ] Provider comparison metrics

## Dependencies

- All Sprint 1-3 AI features operational

## Learning Objectives

- AI-specific observability patterns
- Token economics and cost optimization
- OpenTelemetry for AI workloads
- Production monitoring for LLM applications
