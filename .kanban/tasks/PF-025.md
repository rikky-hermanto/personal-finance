## Objective

Cache LLM responses for semantically similar queries. "How much on food in Jan?" and "January food spending?" should hit the same cache entry.

## Acceptance Criteria

- [ ] Query embedding + similarity comparison before LLM call
- [ ] Cache hit threshold configurable (e.g., cosine similarity > 0.95)
- [ ] Cache storage in PostgreSQL (pgvector for similarity, JSONB for response)
- [ ] Cache invalidation when underlying data changes
- [ ] Cache hit rate metrics
- [ ] TTL-based expiration

## Dependencies

- [PF-018](PF-018.md) — Embedding generation working

## Learning Objectives

- Semantic caching vs exact caching
- Embedding similarity for cache matching
- Cache invalidation strategies for AI
- Cost reduction patterns for LLM applications
