## Objective

Write unit tests for all bank statement parsers (`BcaCsvParser`, `DefaultCsvParser`, `NeoBankPdfParser`) and the bank auto-detection logic (`BankIdentifier`). Parsers currently have zero test coverage.

## Acceptance Criteria

- [ ] `BcaCsvParserTests` — happy path parses correct date/description/amount/flow, CR→Income/DB→Expense mapping, stops at `Saldo Awal`, missing header throws `InvalidDataException`, invalid amount throws `FormatException`
- [ ] `DefaultCsvParserTests` — happy path, malformed rows skipped, empty file returns empty list
- [ ] `BankIdentifierTests` — BCA CSV identified correctly, NeoBank PDF identified correctly, unknown file returns null
- [ ] `NeoBankPdfParserTests` — if no real PDF fixture available, at minimum test empty/corrupt stream handling
- [ ] Mock `ICategoryRuleService` returns predictable categories for all parser tests (do not test categorization here)
- [ ] Test CSV content provided as `MemoryStream` from inline strings — no file I/O
- [ ] `dotnet test` passes with all new tests green

## Technical Notes

- BCA CSV test fixture (inline string):
```
Date,Reference,Description,Branch,...

02/01/2025,ATM WITHDRAWAL,,50000.00,DB,4950000.00
```
