## Objective

Connect .NET API to Python AI service. Upload PDF via .NET endpoint, forward to Python for AI parsing, return structured JSON. Two services talking. Maps to **Day 5** of the learning schedule.

## Acceptance Criteria

- [ ] .NET endpoint `POST /api/statements/upload` accepts PDF
- [ ] .NET forwards PDF bytes to Python service via HttpClient
- [ ] Python service processes and returns structured JSON
- [ ] .NET receives and returns the parsed result to client
- [ ] Docker Compose updated: .NET + Python + PostgreSQL all running
- [ ] Service discovery via Docker networking (not localhost)

## Technical Notes

```
Flow: Browser → .NET API (7208) → Python AI (8000) → Anthropic API → back
```

- Use `IHttpClientFactory` in .NET for resilient HTTP calls
- Add Python service to `docker-compose.yml`
- Environment variable for Python service URL
- Consider timeout handling (LLM calls can be slow)
