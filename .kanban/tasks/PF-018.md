## Objective

Enable pgvector extension on PostgreSQL and build an embedding pipeline for transaction data. Foundation for semantic search and RAG.

## Acceptance Criteria

- [ ] pgvector extension enabled in PostgreSQL container
- [ ] EF Core migration adds vector column to transactions
- [ ] Embedding generation service (OpenAI text-embedding-3-small or local Ollama)
- [ ] Transactions embedded on import (description + metadata combined)
- [ ] Batch re-embedding endpoint for existing transactions
- [ ] Vector dimension configurable (1536 for OpenAI, 768 for local)

## Learning Objectives

- What are embeddings and why they matter for search
- pgvector: vector storage, indexing, distance functions
- Embedding model selection and trade-offs
- Chunking and combining metadata for better embeddings
