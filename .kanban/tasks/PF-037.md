## Objective

Write integration tests for `TransactionsController` and `CategoryRulesController` using ASP.NET Core's `WebApplicationFactory`. Tests exercise the full HTTP pipeline (routing, middleware, serialization) with an in-memory database — no Docker or live DB needed.

## Acceptance Criteria

- [ ] `WebApplicationFactory<Program>` configured with in-memory DB replacing PostgreSQL
- [ ] `TransactionsController` — `GET /api/transactions/health` returns 200 `{"status":"Healthy"}`
- [ ] `TransactionsController` — `GET /api/transactions` returns 200 array
- [ ] `TransactionsController` — `GET /api/transactions/{id}` returns 404 for unknown id
- [ ] `TransactionsController` — `POST /api/transactions/submit` with valid body returns 200 with count message
- [ ] `TransactionsController` — `POST /api/transactions/upload-preview` with empty file returns 400
- [ ] `TransactionsController` — `POST /api/transactions/upload-preview` with unsupported content type returns 400
- [ ] `TransactionsController` — `GET /api/transactions/aggregated` returns 200 with Summary/CurrentMonth/CashFlow shape
- [ ] `CategoryRulesController` — `GET /api/categoryrules` returns 200 array
- [ ] `CategoryRulesController` — `POST /api/categoryrules` with valid body returns 200 with id assigned
- [ ] `CategoryRulesController` — `PUT /api/categoryrules/{id}` non-existent returns 404
- [ ] `CategoryRulesController` — `DELETE /api/categoryrules/{id}` non-existent returns 404
- [ ] `dotnet test` passes

## Technical Notes

- Add `Microsoft.AspNetCore.Mvc.Testing` NuGet to test project
- Override `ConnectionStrings__Default` in `WebApplicationFactory` with in-memory provider:
```csharp
services.RemoveAll(typeof(DbContextOptions<AppDbContext>));
services.AddDbContext<AppDbContext>(o => o.UseInMemoryDatabase("IntegrationTest"));
```
- Use `HttpClient` from factory for all requests
- Test class: `tests/PersonalFinance.Tests/Integration/TransactionsControllerTests.cs`
- Test class: `tests/PersonalFinance.Tests/Integration/CategoryRulesControllerTests.cs`
- `Program.cs` may need `public partial class Program {}` appended for `WebApplicationFactory` to access it

## Dependencies

- None (in-memory DB, no live infrastructure)

## Notes

_These tests catch routing bugs, serialization mismatches, and middleware issues that unit tests miss. Critical before adding AI endpoints._
