## Objective

Wrap the LLM parsing logic into a FastAPI service with proper request/response models. This becomes the Python AI service in the architecture. Maps to **Day 3** of the learning schedule.

## Acceptance Criteria

- [ ] FastAPI application with `/health` and `POST /parse` endpoints
- [ ] Pydantic request model: `{ text: str, bank_hint?: str }`
- [ ] Pydantic response model: `{ transactions: list[TransactionResult] }`
- [ ] LLM call integrated into the endpoint
- [ ] Proper error handling (LLM failures, invalid input)
- [ ] Structured JSON logging matching .NET side format
- [ ] Runs via `uvicorn` on port 8000
- [ ] Dockerfile for the service

## Technical Notes

