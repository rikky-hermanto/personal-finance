## Objective

Define .NET API endpoints as tools that the LLM can call. The LLM decides which API to invoke based on user intent.

## Acceptance Criteria

- [ ] Tool definitions for: list transactions, filter by date/category, get aggregates
- [ ] LLM receives tool definitions and can select appropriate ones
- [ ] Tool execution: LLM output → API call → result back to LLM
- [ ] Multi-tool scenarios: LLM chains multiple API calls
- [ ] Error handling when tool execution fails
