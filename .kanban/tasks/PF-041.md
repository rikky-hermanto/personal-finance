## Objective

Set up Playwright as the E2E / functional testing framework. Cover the core user journeys: upload a bank statement, preview transactions, submit, and verify they appear on the dashboard. Tests run against the full stack (frontend + .NET API + DB via Docker Compose).

**Why Playwright over Cypress:**
- First-class TypeScript (matches the project stack)
- Auto-wait by default — no manual waits/retries
- Can test the .NET API directly via `request` fixture (no browser needed for API assertions)
- Multi-browser: Chromium, Firefox, WebKit in one run
- Codegen (`npx playwright codegen`) to record tests from real interaction

## Technical Notes

### Install
```bash
npm install -D @playwright/test
npx playwright install --with-deps chromium
```

### `playwright.config.ts` skeleton
```ts
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './e2e',
  fullyParallel: true,
  retries: process.env.CI ? 2 : 0,
  use: {
    baseURL: 'http://localhost:8080',
    trace: 'on-first-retry',
  },
  projects: [
    { name: 'chromium', use: { ...devices['Desktop Chrome'] } },
  ],
  webServer: {
    command: 'npm run dev',
    url: 'http://localhost:8080',
    reuseExistingServer: true,
  },
});
```

### Test file naming
- `e2e/health.spec.ts` — API health (no browser)
- `e2e/dashboard.spec.ts` — dashboard render
- `e2e/upload.spec.ts` — upload + submit flow
- `e2e/category-rules.spec.ts` — CRUD

### Running against Docker stack
```bash
docker compose up -d          # start full stack
npx playwright test           # run all E2E tests
```

### API fixture example (no browser)
```ts
test('health check returns Healthy', async ({ request }) => {
  const res = await request.get('http://localhost:7208/api/transactions/health');
  expect(res.ok()).toBeTruthy();
  const body = await res.json();
  expect(body.status).toBe('Healthy');
});
```
