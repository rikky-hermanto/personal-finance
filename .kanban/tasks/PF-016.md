## Objective

Use LLM to automatically categorize transactions that don't match existing keyword rules. Enhance the existing rule-based categorization with AI fallback.

## Acceptance Criteria

- [ ] Transactions without a category rule match are sent to LLM
- [ ] LLM suggests category based on description, amount, and context
- [ ] Confidence score returned with each suggestion
- [ ] User can accept/reject AI category suggestions
- [ ] Accepted suggestions optionally create new category rules
- [ ] Batch categorization endpoint for bulk uncategorized transactions
