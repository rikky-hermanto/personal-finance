## Objective

Fix a second N+1 query pattern in `TransactionsController.SubmitTransactions`: `_categoryRuleService.GetAllAsync()` is called inside a `foreach` loop over every submitted transaction, generating one DB query per transaction.

## Bug Details

**File:** `api/src/PersonalFinance.Api/Controllers/TransactionsController.cs` line 114

**Current code (broken):**
```csharp
foreach (var dto in transactions)
{
    if (!string.IsNullOrWhiteSpace(dto.Category))
    {
        var existing = (await _categoryRuleService.GetAllAsync())  // DB query per loop iteration!
            .Any(r => r.Category == dto.Category && r.Type == dto.Type);
        ...
    }
}
```

**Impact:** Submitting 100 transactions = 100 `SELECT * FROM category_rules` queries.

## Acceptance Criteria

- [ ] `GetAllAsync()` called exactly once before the loop
- [ ] In-loop check uses the pre-fetched in-memory list
- [ ] Behavior is identical: new categories still create rules, existing ones do not
- [ ] `dotnet build` succeeds
- [ ] `dotnet test` passes

## Technical Notes

**Fix:**
```csharp
var existingRules = await _categoryRuleService.GetAllAsync();  // once, before loop

foreach (var dto in transactions)
{
    if (!string.IsNullOrWhiteSpace(dto.Category))
    {
        var existing = existingRules.Any(r => r.Category == dto.Category && r.Type == dto.Type);
        if (!existing && dto.CategoryRuleDto is not null)
            await _categoryRuleService.AddAsync(dto.CategoryRuleDto);
    }
}
```
