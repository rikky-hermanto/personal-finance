## Objective

Build a Retrieval-Augmented Generation pipeline: embed the user's question, find similar transactions via vector search, inject as context, LLM generates answer.

## Acceptance Criteria

- [ ] Query embedding: user question → vector
- [ ] Vector similarity search against transaction embeddings (top-k)
- [ ] Context construction: retrieved transactions formatted as LLM context
- [ ] LLM generates answer grounded in retrieved data
- [ ] Citations: response references specific transactions
- [ ] Configurable top-k and similarity threshold

## Learning Objectives

- RAG architecture: retrieve → augment → generate
- Vector similarity search (cosine distance)
- Context window management
- Grounding and citation patterns
- RAG vs fine-tuning trade-offs
