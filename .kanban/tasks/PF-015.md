## Objective

Create a provider abstraction so the system can switch between Anthropic, OpenAI, and Ollama without code changes. Configuration-driven LLM selection.

## Acceptance Criteria

- [ ] `ILLMProvider` interface with `parse()`, `complete()` methods
- [ ] `AnthropicProvider` implementation (primary)
- [ ] `OpenAIProvider` implementation (fallback)
- [ ] `OllamaProvider` implementation (local dev / cost saving)
- [ ] Provider selected via environment variable
- [ ] Fallback chain: primary fails → secondary attempted
- [ ] Token usage logged per request regardless of provider
