## Objective

Add PDF upload to the FastAPI service. Extract raw text from bank statement PDFs using PyMuPDF, then chain with LLM for structured parsing. Maps to **Day 4** of the learning schedule.

## Acceptance Criteria

- [ ] `POST /parse-pdf` endpoint accepts PDF file upload
- [ ] PyMuPDF extracts text from all pages
- [ ] Extracted text is sent to LLM for structured parsing
- [ ] End-to-end chain: PDF upload → text extraction → LLM → structured JSON
- [ ] Handles multi-page PDFs
- [ ] Graceful error for corrupted/password-protected PDFs

## Technical Notes

- Use `pymupdf` (fitz) for PDF text extraction
- The raw text will be messy — that's expected, the LLM cleans it up
- Consider chunking strategy for large PDFs (token limits)
- Test with real Indonesian bank statement PDFs (BCA, Neo)

## Dependencies

## Learning Objectives

- PDF parsing techniques and challenges
- Chaining multiple processing steps (extract → clean → parse)
- Handling LLM token limits with chunking
- File upload handling in FastAPI
